# 📏 Замеры тела — Telegram Mini App

> **🚀 Развёрнуто:** https://olgarad2025.github.io/body-tracker/
> Репозиторий: https://github.com/olgarad2025/body-tracker

Приложение для отслеживания параметров тела: вес, объём груди, под грудью, талия,
живот, бёдра, бедро левое и правое. По каждому параметру строится график с
**линией тренда** (линейная регрессия) и подписью, куда движется показатель.

- **Хранилище:** Telegram CloudStorage — данные лежат в аккаунте пользователя и
  синхронизируются между устройствами. Сервер и база данных не нужны.
- **Хостинг:** любой статический HTTPS-хостинг (инструкция для GitHub Pages ниже).
- **Стек:** чистый HTML/CSS/JS + [Chart.js](https://www.chartjs.org/) (без сборки).

## Структура

```
body-tracker/
├── index.html          # разметка и подключение скриптов
├── .nojekyll           # чтобы GitHub Pages не обрабатывал файлы Jekyll
├── css/style.css       # стили + поддержка темы Telegram
└── js/
    ├── metrics.js      # список параметров тела
    ├── stats.js        # линейная регрессия и описание тренда
    ├── storage.js      # CloudStorage (в браузере — localStorage)
    ├── charts.js       # построение графика с линией тренда
    └── app.js          # логика интерфейса и Telegram Web App
```

## Локальная проверка в браузере

Вне Telegram приложение работает и хранит данные в `localStorage` (тема — тёмная
по умолчанию). Запусти любой статический сервер:

```bash
cd body-tracker
python -m http.server 8080
# открой http://localhost:8080
```

Кнопки «Сохранить» (MainButton Telegram) в браузере не будет — сохраняй через
консоль или просто открой в Telegram. Для полноценной проверки лучше сразу
задеплоить и открыть в приложении Telegram.

## Деплой на GitHub Pages

1. Создай репозиторий на GitHub и залей туда **содержимое папки `body-tracker`**
   (файлы `index.html`, `css/`, `js/`, `.nojekyll` должны лежать в корне репозитория):

   ```bash
   cd body-tracker
   git init
   git add .
   git commit -m "Body tracker mini app"
   git branch -M main
   git remote add origin https://github.com/<логин>/<репозиторий>.git
   git push -u origin main
   ```

2. В репозитории: **Settings → Pages → Build and deployment**.
   - Source: **Deploy from a branch**
   - Branch: **main**, папка **/(root)** → **Save**.

3. Через минуту сайт будет доступен по адресу
   `https://olgarad2025.github.io/body-tracker/`. Проверь, что он открывается по HTTPS.

## Подключение к боту (BotFather)

1. Открой [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя и username.
   Сохрани токен (для Mini App он не нужен в коде, но пусть будет).

2. Привяжи Mini App к боту одним из способов:

   **Через кнопку меню (проще всего):**
   ```
   /setmenubutton
   → выбери бота
   → пришли URL: https://olgarad2025.github.io/body-tracker/
   → задай текст кнопки, например: Замеры
   ```

   **Или как отдельное Mini App:**
   ```
   /newapp
   → выбери бота
   → заполни название, описание, иконку
   → Web App URL: https://olgarad2025.github.io/body-tracker/
   ```

3. Открой бота в Telegram, нажми кнопку меню (или ссылку приложения) — Mini App
   запустится. Первый замер сохранится в твой CloudStorage.

## Как пользоваться

- **➕ Замер** — выбери дату (по умолчанию сегодня), заполни нужные поля (можно не
  все) и нажми **«Сохранить замер»**. Повторное сохранение за ту же дату перезапишет
  значения; уже сохранённые значения подставляются в форму.
- **📈 Графики** — переключай параметры чипами сверху. Пунктирная линия — тренд;
  бейдж справа показывает направление и скорость (например «↓ 0.4 кг/нед»). Ниже —
  текущее значение, суммарное изменение и диапазон мин–макс.
- **📋 История** — список всех замеров от новых к старым, с возможностью удаления.

## Добавить или переименовать параметр

Отредактируй массив `METRICS` в [`js/metrics.js`](js/metrics.js) — везде в
интерфейсе и на графиках изменения подхватятся автоматически.
